## Where octopus-kb wins

- Returns canonical decisions directly; grep forces the caller to parse match lines.
- Drift detection: grep cannot inspect audit source hashes and scores 0.0 on drift tasks.
- Relationship trace: octopus-kb returns normalized `related_paths` from graph context; grep only sees exact strings.

## Where grep wins / stays competitive

- Raw string search on prose with no frontmatter structure.
- Zero setup cost: no schema, lint, canonical resolution, or audit conventions required.
- Exact-title fact lookup can match if the expected page itself contains the query.

## Reproduction

The baseline is bit-identical because all paths run in Python: no shell grep and no platform-specific binaries. Run:

```bash
octopus-kb eval run --tasks eval/tasks.yaml --out /tmp/my-run
diff -r --exclude='*.metrics.json' eval/runs/2026-04-18-baseline /tmp/my-run
```

Expected: empty diff for `<task_id>.json` and `summary.md`. `*.metrics.json` files vary by run because they contain latency measurements and are excluded.
